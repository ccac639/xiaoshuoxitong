## 架构改造 (22:30 - 23:00)

### 用户反馈（核心问题）
用户指出系统架构的 3 个核心问题：
1. **Skill 权力过大**（变成"第二个写作AI"）
2. **Event Flow 和 Writer 边界不清**（在抢"剧情控制权"）
3. **World State 更新应该是"确认驱动"**（不是"结果驱动"）

**正确架构：**
```
Event Engine（发生什么）
  ↓
Chapter Writer（如何表达）
  ↓
Skill Engine（是否合格）← 只做裁判，不做选手
  ↓
✓ PASS 才进入
  ↓
World Update + Memory Update
```

### 改造内容
1. **Skill 降级**（裁判，不是选手）
   - 修改 `fanqieSkill.ts`：
     - 明确 Skill 定位（只做审计，不做修复）
     - 移除 `suggestions` 字段（Skill 不提供修复建议）
     - 简化 `trackHooks` 函数（伏笔管理不应该由 Skill 做）
   - 更新 `AuditResult` 类型定义（移除 `suggestions` 字段）

2. **World Update gate**（只有 PASS 才更新世界状态）
   - 修改 `generate-chapter` API：
     - 把"更新 World State"步骤移到"审计"之后
     - 只有审计通过（PASS）才更新世界状态
     - 加强 gate（`shouldUpdateWorldState` 字段）

3. **简化 Event Flow**（使用结构化剧情指令）
   - 修改 `generate-chapter` API：
     - 事件类型匹配（`WorldEvent` 类型）
     - 使用模拟数据（快速验证流程）

4. **前端更新**
   - 更新 `AuditPanel.tsx`（移除修改建议区域）
   - 更新 `ChapterView.tsx`（修复类型错误）
