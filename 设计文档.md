# WinkNovel UI 仿制系统设计文档

## 1. 系统概述

### 1.1 项目背景
WinkNovel 是一款基于 AI 的小说创作工具，提供大纲撰写、章节续写、文本润色等功能。本项目的目标是仿制 WinkNovel 的完整 UI 层，包括所有 UI 功能。

### 1.2 设计目标
- 高度还原原版 WinkNovel 的 UI 设计
- 实现完整的前端交互功能
- 支持深色/浅色/国风三种主题切换
- 提供完整的小说创作工作流

### 1.3 技术栈
- **前端框架**: 原生 JavaScript（无框架依赖）
- **样式**: CSS3  with CSS 变量
- **数据存储**: LocalStorage
- **构建工具**: 无（可直接在浏览器中运行）

## 2. 功能模块设计

### 2.1 主页（Home）
**功能描述**: 展示产品介绍和快速入口

**UI 元素**:
- Hero 区域：动态渐变光晕背景
- 功能介绍卡片：6 个主要功能入口
- 快速操作按钮：「开始创作」、「配置 AI」

**交互设计**:
- 点击「开始创作」→ 跳转到书架页面
- 点击「配置 AI」→ 跳转到 AI 配置页面
- 功能卡片点击 → 跳转到对应功能页面

### 2.2 书架（Bookshelf）
**功能描述**: 管理用户的小说列表

**UI 元素**:
- 分栏标签：原创 / 续写
- 小说卡片网格布局
- 新建小说按钮
- 视图切换按钮（网格/列表）

**交互设计**:
- 点击小说卡片 → 进入小说详情页
- 点击「删除」按钮 → 弹出确认删除对话框
- 点击「新建小说」→ 弹出创建小说模态框

**数据结构**:
```javascript
Novel {
  id: number,
  name: string,
  display_name: string,
  chapter_count: number,
  source: 'original' | 'deconstruct_continue',
  writing_style: string,
  created_at: string,
  chapters: Chapter[]
}
```

### 2.3 小说详情（Novel Detail）
**功能描述**: 展示小说详情和章节管理

**UI 元素**:
- 小说标题和元信息
- 标签页：大纲 / 章节 / 实体 / 伏笔
- 章节列表
- 新建章节按钮

**交互设计**:
- 点击章节 → 进入章节编辑器
- 点击「编辑」按钮 → 进入章节编辑器
- 点击「新建章节」→ 弹出创建章节模态框
- 章节状态标识：已写、已润色

**数据结构**:
```javascript
Chapter {
  chapter_number: number,
  title: string,
  has_content: boolean,
  has_refine: boolean,
  content: string
}
```

## 4. 技术架构设计

### 4.1 应用架构

#### 4.1.1 状态管理
使用中央状态对象管理应用状态：
```javascript
const state = {
    currentView: 'home',       // 当前视图
    novels: [],                // 小说列表
    currentNovel: null,       // 当前选中的小说
    currentChapter: null,      // 当前选中的章节
    theme: 'dark',            // 当前主题
    // ... 其他状态
};
```

#### 4.1.2 渲染机制
使用原生 JavaScript 的 DOM API 进行渲染：
- `createElement()`: 创建 DOM 元素
- `render()`: 主渲染函数，根据状态重新渲染整个应用

#### 4.1.3 事件处理
使用事件监听器处理用户交互：
- `addEventListener()`: 添加事件监听器
- 事件委托：在动态创建的元素上添加事件监听器

### 4.2 数据持久化

#### 4.2.1 LocalStorage 结构
```javascript
{
  "theme": "dark",
  "winknovel_novels": [
    {
      "id": 1,
      "name": "星辰变",
      "display_name": "星辰变",
      "chapter_count": 3,
      "source": "original",
      "writing_style": "默认",
      "created_at": "2026-01-15",
      "chapters": [...]
    }
  ]
}
```

#### 4.2.2 数据操作
- `loadNovels()`: 从 LocalStorage 加载数据
- `saveNovels()`: 保存数据到 LocalStorage
- `createNovel()`: 创建新小说
- `deleteNovel()`: 删除小说
- `createChapter()`: 创建新章节
- `deleteChapter()`: 删除章节
- `saveChapterContent()`: 保存章节内容

### 4.3 主题系统

#### 4.3.1 CSS 变量
使用 CSS 变量定义主题：
```css
:root {
    --bg-base: #070b14;
    --bg-surface: #0e1422;
    --text-primary: #e8eef8;
    --accent: #4f8cff;
    /* ... 其他变量 */
}
```

#### 4.3.2 主题切换
通过修改 `document.documentElement.setAttribute('data-theme', theme)` 切换主题

## 5. 交互设计

### 5.1 页面导航
- **顶部导航标签**: 主页、书架、续写、润色、AI配置、提示词、请求记录
- **返回按钮**: 在详情页和编辑器中显示
- **面包屑**: 无（使用返回按钮代替）

### 5.2 模态框交互
- **打开**: 点击触发按钮 → 显示模态框
- **关闭**: 点击遮罩层 或 点击取消按钮
- **确认操作**: 点击确认按钮 → 执行操作 → 关闭模态框

### 5.3 主题切换交互
- **打开**: 点击主题切换按钮 → 显示主题选择面板
- **切换**: 点击主题选项 → 立即切换主题
- **关闭**: 点击面板外区域 → 关闭主题选择面板

### 5.4 通知提示
- **显示**: 执行操作后显示 Toast 通知
- **自动消失**: 2 秒后自动消失
- **动画**: 从底部滑入

## 6. 响应式设计

### 6.1 断点
- **桌面**: > 768px
- **移动**: ≤ 768px

### 6.2 适配策略
- **导航标签**: 移动端横向滚动
- **网格布局**: 自适应列数
- **润色工作室**: 移动端单列布局
- **表格**: 移动端改为卡片布局
