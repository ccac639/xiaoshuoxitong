## 📂 目录说明

### 1. `public/` - 静态资源目录
存放不需要经过构建工具处理的静态文件，如 `index.html` 入口、字体文件、favicon 等。

### 2. `src/` - 源代码目录
项目的核心代码都放在这里。

#### 2.1 `src/assets/` - 资源文件
- **images/**: 图片、图标等视觉资源
- **styles/**: 全局样式、CSS 变量、重置样式

#### 2.2 `src/components/` - 组件目录
按组件类型分层：
- **common/**: 通用基础组件（按钮、输入框、模态框等）
- **layout/**: 布局组件（页头、侧边栏、底部栏）
- **theme/**: 主题相关组件

#### 2.3 `src/pages/` - 页面级组件
每个页面一个独立文件夹，包含该页面的所有相关文件：
- **home/**: 主页
- **library/**: 书架页面
- **novel-detail/**: 小说详情页面
- **editor/**: 章节编辑器
- **polish/**: 润色工作室
- **continue/**: 续写页面
- **ai-config/**: AI 配置页面
- **prompts/**: 提示词管理
- **trace/**: 请求记录

#### 2.4 `src/services/` - 服务层
- **storage.js**: 本地存储封装
- **api.js**: API 调用（模拟）
- **export.js**: 导出功能

#### 2.5 `src/utils/` - 工具函数
纯函数工具库，可在多处复用。

#### 2.6 `src/hooks/` - 自定义 Hooks
（适用于 React 项目）封装可复用的状态逻辑。

#### 2.7 `src/store/` - 状态管理
（如果使用 Redux/Vuex）集中管理应用状态。

### 3. `styles/` - 独立样式目录
如果不使用 CSS-in-JS 或 CSS Modules，可以将样式文件统一放在这里。

### 4. `docs/` - 文档目录
存放项目的所有设计文档、开发文档、API 文档等。

### 5. `tests/` - 测试目录
- **unit/**: 单元测试
- **integration/**: 集成测试
- **e2e/**: 端到端测试

## 📦 模块化设计原则

### 1. 单一职责原则 (SRP)
每个文件/组件只负责一个功能。

**示例：**
```javascript
// ❌ 错误：一个文件包含多个不相关功能
// UserAndNovelManager.js
export const createUser = () => { ... }
export const deleteNovel = () => { ... }

// ✅ 正确：分离到不同文件
// user.js
export const createUser = () => { ... }

// novel.js
export const deleteNovel = () => { ... }
```

### 2. 高内聚低耦合
相关功能放在同一目录下，减少跨模块依赖。

**示例：**
```
pages/novel-detail/
├── NovelDetailPage.js    # 页面主组件
├── ChapterList.js        # 章节列表（仅在此页面使用）
├── ChapterItem.js        # 章节项（仅在此页面使用）
└── novel-detail.css      # 页面样式
```

### 3. 可复用组件提升
如果某个组件在 3 个以上地方使用，提升为通用组件。

**示例：**
```
// 最初：在特定页面内定义
pages/library/Button.js

// 复用 3 次后：提升为通用组件
components/common/Button.js
```

## 🔄 数据流向设计

### 单向数据流
```
User Action → Component → Hook/Handler → Service → Storage
                ↓
                └────────── State Update ←──────────┘
                               ↓
                          Re-render UI
```

### 状态管理策略
| 状态类型 | 存储位置 | 示例 |
|---------|---------|------|
| 全局 UI 状态 | Store / Context | 当前主题、侧边栏展开状态 |
| 页面级状态 | 页面组件 state | 当前选中的标签页 |
| 组件级状态 | 组件 state | 输入框内容、下拉菜单展开 |
| 持久化数据 | localStorage | 小说数据、API 配置 |
| 临时数据 | 内存 (变量) | 未保存的编辑器内容 |

## 🎨 样式组织策略

### 方案 A：CSS Modules（推荐）
```javascript
// Button.js
import styles from './Button.module.css';

function Button({ children }) {
  return <button className={styles.button}>{children}</button>;
}
```

### 方案 B：CSS-in-JS（如 styled-components）
```javascript
// Button.js
import styled from 'styled-components';

const StyledButton = styled.button`
  padding: 8px 16px;
  border-radius: 6px;
`;

function Button({ children }) {
  return <StyledButton>{children}</StyledButton>;
}
```

### 方案 C：传统 CSS 文件
```javascript
// Button.js
import './Button.css';

function Button({ children }) {
  return <button className="btn">{children}</button>;
}
```

## 📄 文件命名规范

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| 页面组件 | PascalCase + Page 后缀 | `HomePage.js` |
| 通用组件 | PascalCase | `Button.js`, `Modal.js` |
| 工具函数 | camelCase | `helpers.js`, `validators.js` |
| 样式文件 | kebab-case | `main.css`, `home-page.css` |
| 配置文件 | camelCase | `config.js`, `constants.js` |
| 测试文件 | 原文件名 + .test | `Button.test.js` |
